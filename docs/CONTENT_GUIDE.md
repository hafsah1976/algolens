# AlgoLens Content Guide

Use this guide when adding topics, lessons, quizzes, Trace Mode frames, or coding problems.

## Voice

- Explain the state change before the abstraction.
- Prefer "what changed?" over "what is the formal definition?"
- Use short sentences and concrete examples.
- Avoid unexplained jargon. If a term is necessary, define it nearby.
- End lessons with a stopping cue or recap instead of sending learners into another feed.

## Lesson structure

Each lesson should answer:

- What problem is this pattern solving?
- What data is changing over time?
- Which variable, pointer, map entry, stack item, or node matters right now?
- Why is the next move allowed?
- What beginner mistake should the learner avoid?

## Trace Mode frame structure

Each frame should include:

- A clear title
- Current visual state
- Active code line or pseudocode step
- Beginner explanation
- Variables or data-state changes
- One decision or takeaway
- Optional active-recall prediction

## Quiz rules

- Ask about concepts, not trivia.
- Explain why the answer is correct after submission.
- Keep wrong options plausible but not mean-spirited.
- Prefer questions that reinforce Trace Mode mental models.

## Coding problem rules

- Keep starter problems small.
- Include at least one public test and one hidden test.
- State constraints in plain language.
- Keep starter code focused on the function students need to write.

## Beginner-friendly rewrite checklist

Before publishing content, ask:

- Can a student picture the data structure?
- Does the lesson explain why a move is safe?
- Are key terms defined?
- Is there a concrete example before the general rule?
- Does the learner know when to stop and recap?
